Aanpak

Aan de hand van de verschillende user stories ben ik verschillende functionaliteiten gaan programmeren in de Simac app. Een aantal user stories waar ik mee te werk ben gegaan zijn hieronder te vinden:

- Als Simac werknemer wil ik een recept uit de Simac app opzoeken en klaar gaan maken.
- Als Simac werknemer wil ik kunnen zien welk recept goed gewaardeerd wordt, zodat ik weet welk recept lekker is.
- Als Simac werknemer wil ik bestaande recepten kunnen wijzigen, zodat de gegevens kloppen.
- Als Simac werknemer wil ik een bestaande recepten verwijderen.
- Als Simac werknemer wil ik verschillende recepten kunnen opslaan door middel van deze te liken.

Aan de hand van deze user stories ben ik de volgende basis functionaliteiten gaan bouwen voor het kook gedeelte van de Simac app. Zodra deze gebouwd zijn wil ik deze functionaliteiten gaan testen op de doelgroep om de gebruikers ervaring van de Simac app te verbeteren.

Ophalen van recepten

Om alle recepten op te halen van de Simac app heb ik als eerste 2 states aangemaakt genaamd recipes en favorites. Deze hebben als initial value NULL en worden bij het laden van het scherm gevuld met de bijhorende data. Dit wordt gedaan door middel van een useEffect hook waarin ik als eerste een call doe naar de collection recipes in firestore. Hier set ik de recipes state met de data die uit de firestore backend komt. Het zelfde doe ik hierna nog een keer voor de favorieten, hierbij maak ik de call naar de collection users waarin ik de document pak van de gebruiker die is ingelogd. Als laatste kan ik nu de recepten filteren op de favorieten van de gebruiker door de recepten te filteren op recepten waarvan de ID overeenkomt met die van de favorieten.

Toevoegen van recepten

Voor het toevoegen van recepten moet er een afbeelding in de backend worden opgeslagen. Het is bad practice om deze in de firebase backend op te slaan omdat wanneer de app veel gebruikers heeft dit enorm veel opslag en uiteindelijk dus geld kost. Om deze reden sla ik de afbeeldingen op in Amazon S3 Bucket en sla ik de link van de afbeelding op in Firebase. Hiervoor doe ik eerst een call naar firebase om de API keys van mijn S3 bucket op te halen. Deze moeten absoluut niet in de code worden opgeslagen wegens veiligheids redenen.

Vervolgens heb ik een functie gemaakt die wordt aangeroepen wanneer de create button wordt geklikt. Deze checkt eerst of alle input velden zijn ingevuld, zodra dit niet het geval is geeft die een alert met de message dat alle input velden ingevuld moeten zijn. Als dit wel het geval is wordt er een random image id gemaakt, dit is handig voor later wanneer we een recept willen verwijderen of aanpassen. Vervolgens wordt de image compressed naar een klein formaat, dit zorgt ervoor dat gebruikers geen enorme foto's kunnen toevoegen wat onnodig veel opslag kost en vervolgens heel lang duurt voor de gebruiker om te laden. Nadat de image in de S3 bucket is opgeslagen wordt de image url in een variable opgeslagen. Deze wordt samen met de andere data van het aangemaakte recept geupload naar Firebase.

Aanpassen van recepten

Zodra er op een edit recipe button wordt geklikt wordt er via props de gekozen recept info meegegeven naar de functie die wordt aangeroepen. In de onRecipeEditClick functie worden de verschillende states die bij het recept horen gevuld met de data die via de props werd meegegeven. Vervolgens kan de gebruiker deze input velden aanpassen en op save button klikken. door op de save button te klikken wordt de functie onEdit aangeroepen waarin er weer wordt gechecked of alle input velden zijn ingevuld. Zodra dit het geval is wordt de nieuwe data naar het bijhorende recept gestuurd door de call uitevoeren naar collection recipes met als document de ID van het geselecteerde recept.

Verwijderen van recepten

Wanneer de gebruiker op de delete button van een recept klikt wordt de functie onDeleteRecipeClick aangeroepen waardoor de gebruiker een popup te zien krijgt waar wordt gevraagd of hij/zij zeker weet of die het recept wilt verwijderen. Zodra de gebruiker hier op delete klikt word de functie onDelete aangeroepen.

Hier wordt als eerst de afbeelding uit de S3 Bucket verwijderd. Hier komt de imageID weer van pas die we bij het maken van een recept hebben aangemaakt. Hierdoor weten we de url van de afbeelding en kunnen we deze dus gemakkelijk verwijderen. Vervolgens wordt het recept ook in Firebase verwijderd.

5. Recepten favorieten

Op de recept detail pagina wordt via parameters de bijhorende data van het recept meegegeven. Daarnaast is er een state waarin ik met een boolean bijhoud of het recept een favoriet is van de gebruiker. Wannneer deze true is, is het hartje ingekleurd en wanneer deze false is, is deze niet ingekleurd. In een useEffect hook check ik of het recept een favoriet is door te kijken of de id van het recept voorkomt in de favorieten van de gebruiker.

Wanneer de gebruiker op het hartje klikt wordt de like functie aangeroepen, hier wordt gechecked of de like state true or false is. Wanneer deze false is wordt die toegevoegd aan de favorieten van de gebruiker in firebase, waardoor het hartje ook ingekleurd word. Wanneer de state true is en er op het hartje wordt geklikt, wordt de favoriet van de gebruiker verwijderd.
